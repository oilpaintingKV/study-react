# React Basic
## Why React?
### The first reason
- There was too much overlap in the old way.
- Redundancy modifications, also called 'Shotgun Surgery', cause too many resources to be wasted.
- React is a component-based ui library that can solve these problems.

### The second reason
- Imperative Programming vs Declarative Programming
#### Imperative Programming
- jQuery
- HOW?
#### Declarative Programming
- React
- WHAT?
- Declarative programming has been in the spotlight recently.

### The third reason
- virtual DOM
#### DOM?
- Document Object Model
- Rather than applying changes that occur in JavaScript's process of adding elements directly to real elements, it is possible to update the virtual DOM in advance (the rendering process is omitted) and then update it at once.
